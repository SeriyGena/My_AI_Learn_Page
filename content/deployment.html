A free service from GitHub to host static websites directly from your repository. Ideal for single HTML files.

  • **Step 1: Save your HTML file.** Save the entire code from this page into a single file named index.html on your computer. This name is crucial for GitHub Pages to find your main page automatically.
  • **Step 2: Create a GitHub account.** If you don't have one, sign up at github.com.
  • **Step 3: Create a new repository.**
    • Click the "+" icon in the top right corner and select "New repository".
    • Give it a name (e.g., my-ai-course).
    • Make sure it's **Public**.
    • Optionally, check "Add a README file".
    • Click "Create repository".
  • **Step 4: Upload your index.html file.**
    • In your newly created repository, click "Add file" -> "Upload files".
    • Drag and drop your index.html file into the upload area, or click "choose your files" to select it.
    • Scroll down and click "Commit changes" to save the file to your repository.
  • **Step 5: Enable GitHub Pages.**
    • Go to the "Settings" tab in your repository.
    • In the left sidebar, click "Pages".
    • Under "Build and deployment", for "Source", select the branch where your index.html file is located (usually main).
    • Ensure the folder is set to / (root).
    • Click "Save".
  • **Step 6: Get your site link.** GitHub Pages will take a few minutes to build and deploy your site. After it's done, refresh the "Pages" settings. You will see a message like "Your site is live at https://<your_username>.
