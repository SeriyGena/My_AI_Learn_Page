Website Deployment

Learn how to host your interactive course plan online for free and configure data persistence.

1. GitHub Pages üöÄ

A free service from GitHub to host static websites directly from your repository. Ideal for single HTML files.

  1. Save your HTML file. Save the entire code from this page and the other related files (tools.html, resources.html, etc.) into your project folder. The main file must be named index.html.
  2. Create a GitHub account. If you don't have one, sign up at github.com.
  3. Create a new repository. Click the "+" icon in the top right corner and select "New repository". Give it a name (e.g., my-ai-course). Make sure it's Public. Click "Create repository".
  4. Upload your project files. In your newly created repository, click "Add file" -> "Upload files". Drag and drop all your project files and the entire content folder into the upload area. Scroll down and click "Commit changes".
  5. Enable GitHub Pages. Go to the "Settings" tab in your repository. In the left sidebar, click "Pages". Under "Build and deployment", for "Source", select the branch where your files are located (usually main). Ensure the folder is set to / (root). Click "Save".
  6. Get your site link. GitHub Pages will take a few minutes to build and deploy your site. After it's done, refresh the "Pages" settings. You will see a message like "Your site is live at https://<your_username>.github.io/<your_repo_name>/". This is your shareable link!

2. Firebase Setup (for Progress Saving) üîê

This is the most important step for saving and restoring your progress. It requires a free Google account.

  1. Create a Firebase Project. Go to the Firebase Console. Click "Add project" and follow the on-screen instructions. Give your project a name (e.g., "AI Course Tracker"). You don't need Google Analytics for this app.
  2. Register your web app. In your new project, click the web icon </> to register a new web app. Give your app a name and click "Register app".
  3. Get your Firebase Config. After registering, you will see a JavaScript object named firebaseConfig. It contains your unique apiKey, authDomain, and projectId. Copy the entire firebaseConfig object.
  4. Update your index.html. Open your index.html file in a code editor. Find the existing firebaseConfig object in the <script type="module"> tag at the top of the file and replace it with the one you just copied from your Firebase project.
  5. Enable Sign-in Methods. In the Firebase Console, go to Authentication -> Sign-in method tab. Enable both Email/Password and Anonymous providers.
  6. Add Authorized Domain. While still in Authentication settings, add your GitHub Pages domain to the list of authorized domains (e.g., your-username.github.io).
  7. Set up Firestore Rules. Go to Firestore Database -> Rules tab. Replace the default rules with the following to allow authenticated users to access their own data: 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /artifacts/{appId}/users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
    Click Publish. Your app is now fully configured!
